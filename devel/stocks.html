Stocks¶

Overview¶

Access to the Stocks endpoints of the IEX Developer API is available through the top-level Stock function.

Calling this function with a symbol (str) or list of symbols (list) will return a StockReader instance, which allows retrieval of endpoints (Earnings, Quote, etc) for up to 100 symbols at once. There are three ways to access such endpoints:

1. Endpoint Methods - Allow retrieval of individual endpoints. Most endpoints allow Pandas DataFrame formatting. Examples are get_book, get_quote, etc. Where applicable, parameters (i.e. displayPercent for the Quote endpoint) may be passed to these methods as keyword arguments.

2. Field Methods - supported by certain endpoints. Allow quick and lightweight access to select fields of the Quote and Key Stats endpoints. Examples are get_company_name, get_beta, etc. See below for more examples.

3. get_endpoints - returns one or more (up to 10) endpoints to be returned in the exact format of the examples in the IEX docs. This method accepts no additional parameters (it uses the defaults) and does not allow Pandas DataFrame as an output format.

In [1]: from iexfinance.stocks import Stock

In [2]: aapl = Stock("aapl")

In [3]: aapl.get_price()
Out[3]: 180.94

Formatting¶

Supported Formats¶

1. JSON (All Endpoints) - For a single symbol, endpoint methods will return the endpoint as specified in docs (i.e. dict for Quote, float for Price) while multiple symbols will be returned as a symbol-indexed dictionary of the requested endpoint for each symbol.

2. Pandas DataFrame (most endpoints) - The DataFrame is often indexed by the endpoint’s dictionary keys, with a column for each requested symbol. For field methods, the DataFrame is a single column, indexed by each requested symbol.

Selecting an Output Format¶

Output formatting is most often selected through the initial call to the Stock function (which becomes an attribute of the StockReader instance) by passing output_format as a keyword argument:

In [4]: from iexfinance.stocks import Stock

In [5]: aapl = Stock("aapl", output_format='pandas')

In [6]: aapl.get_quote().head()
Out[6]: 
                      AAPL
avgTotalVolume    42161798
calculationPrice     close
change                 6.7
changePercent      0.03845
close               180.94

It is also possible to change the output format of an already-instantiated StockReader:

In [7]: aapl.output_format = 'json'

In [8]: aapl.get_quote()["close"]
Out[8]: 180.94

Further, it is possible to customize the integer and floating point JSON parsing (into a type such as Decimal.decimal) by passing the desired types via json_parse_float and json_parse_int parameters.

In [9]: from decimal import Decimal

In [10]: aaplD = Stock("AAPL", json_parse_float=Decimal)

Parameters¶

Certain endpoints (such as quote and chart) allow customizable parameters. To specify one of these parameters, merely pass it as a keyword argument to the endpoint method. (see example).

In [11]: aapl = Stock("AAPL", output_format='pandas')

In [12]: aapl.get_quote(displayPercent=True).loc["ytdChange"]
Out[12]: 
AAPL    6.51305
Name: ytdChange, dtype: object

Note

A key IEX optional parameter is filter_, which allows the filtering of endpoint requests to return certain fields only. See the IEX docs and the examples below for more information.

Examples¶

Endpoint Methods¶

A single symbol request will return data exactly as it appears in the IEX docs examples:

In [13]: from iexfinance.stocks import Stock

In [14]: aapl = Stock("AAPL")

In [15]: aapl.get_price()
Out[15]: 180.94

While multi-symbol requests will return a symbol-indexed list of the endpoint’s data

Most endpoints can be formatted as a pandas.DataFrame. Multi-symbol requests will concatenate the dataframes for each:

In [16]: from iexfinance.stocks import Stock as iex

In [17]: air_transport = Stock(['AAL', 'DAL', 'LUV'], output_format='pandas')

In [18]: air_transport.get_quote().head()
Out[18]: 
                       AAL      DAL      LUV
avgTotalVolume    10465379  6781971  6045641
calculationPrice     close    close    close
change                0.65     0.56     0.29
changePercent      0.01698   0.0096  0.00549
close                38.94    58.87    53.08

Filtering¶

Per the IEX Docs, the filter, paramter may be passed to any endpoint request to restrict single endpoint queries to certain fields. iexfinance uses filter_, as filter is a reserved word in Python:

In [19]: from iexfinance.stocks import Stock as iex

In [20]: aapl = Stock("AAPL", output_format='pandas')

In [21]: aapl.get_quote(filter_='ytdChange')
Out[21]: 
              AAPL
ytdChange  0.06513

Lists of fields are acceptable as well:

In [22]: aapl.get_quote(filter_=['ytdChange', 'open', 'close'])
Out[22]: 
                AAPL
close      180.94000
open       176.74000
ytdChange    0.06513

Note

The desired fields must exactly match the field key names as listed in the IEX docs.

Passing Parameters¶

Endpoint-specific

We show an example using the last parameter for the News endpoint:

In [23]: aapl = Stock("AAPL")

In [24]: len(aapl.get_news())
Out[24]: 10

by default, News returns the last 10 items (last is 10 by default), but we can specify a custom value:

In [25]: aapl = Stock("AAPL")

In [26]: len(aapl.get_news(last=35))
Out[26]: 15

With a custom value specified, News now returns the previous 35 items.

Output Formatting¶

Most endpoints allow for pandas DataFrame-formatted output:

In [27]: aapl = Stock("AAPL", output_format='pandas')

In [28]: aapl.get_quote().head()
Out[28]: 
                      AAPL
avgTotalVolume    42161798
calculationPrice     close
change                 6.7

We can also change the output format once our StockReader object has been instantiated:

In [29]: aapl.output_format == 'json'
Out[29]: False

In [30]: aapl.get_ohlc()
Out[30]: 
                                           AAPL

Cryptocurrencies¶

As of the 5/18/2018 IEX Provider update, quotes are provided for certain Cryptocurrencies. Access to these quotes is available by creating a Stock object and using the get_quote method.

The following tickers are supported:

  • Bitcoin USD (BTCUSDT)
  • EOS USD (EOSUSDT)
  • Ethereum USD (ETHUSDT)
  • Binance Coin USD (BNBUSDT)
  • Ontology USD (ONTUSDT)
  • Bitcoin Cash USD (BCCUSDT)
  • Cardano USD (ADAUSDT)
  • Ripple USD (XRPUSDT)
  • TrueUSD (TUSDUSDT)
  • TRON USD (TRXUSDT)
  • Litecoin USD (LTCUSDT)
  • Ethereum Classic USD (ETCUSDT)
  • MIOTA USD (IOTAUSDT)
  • ICON USD (ICXUSDT)
  • NEO USD (NEOUSDT)
  • VeChain USD (VENUSDT)
  • Stellar Lumens USD (XLMUSDT)
  • Qtum USD (QTUMUSDT)
